# Markhand Web upload policy (P0-09)

Status: Done for local policy/sandbox smoke closure. This policy is the upload
security contract for Phase 1B implementation. It does not claim Profile B
malware-scanner coverage.

## 1. Allowlist

Only these canonical formats may enter quarantine for Phase 1B POC conversion:

| Canonical format | Extensions | Required first check |
|---|---|---|
| PDF | `.pdf` | `%PDF-` header and PDF parser page-count preflight |
| DOCX | `.docx` | ZIP container with `[Content_Types].xml` and Word paths |
| PPTX | `.pptx` | ZIP container with `[Content_Types].xml` and PowerPoint paths |
| XLSX/ODS/XLS/XLSB | `.xlsx`, `.ods`, `.xls`, `.xlsb` | Spreadsheet parser preflight |
| CSV | `.csv`, `.tsv` | Text decode and delimiter sniff |
| HTML | `.html`, `.htm` | Text decode and sanitizer parse |
| Plain text | `.txt`, `.md` | Text decode and legacy Vietnamese encoding fallback |
| Image OCR | `.png`, `.jpg`, `.jpeg`, `.webp`, `.tif`, `.tiff`, `.bmp` | Image header and pixel-count preflight |
| Audio | `.wav`, `.mp3`, `.ogg`, `.flac`, `.m4a` | Audio container and duration preflight |

Reject by default:

- Extension or magic-byte spoofing.
- Nested archives not required by the canonical format.
- Active content, scripts, external references, or parser features that require
  network access.
- Any format not listed above.

## 2. Upload limits

All limits are enforced before parser execution where possible and are repeated
inside the sandbox:

- Max upload object size: 200 MiB.
- Max converted output size: 50 MiB.
- Max PDF pages: 500.
- Max image pixels: 80,000,000.
- Max audio duration: 3,600 seconds.
- Max archive entries: 4,096.
- Max archive uncompressed bytes: 1 GiB.
- Max archive compression ratio: 100:1.
- Max per-job wall clock: 420 seconds.
- Max worker CPU: 300 CPU seconds.
- Max worker RAM: 1,536 MiB.
- Max worker process count: 64.
- Max worker open files: 256.

Limit failures are either `reject` at intake/preflight or `quarantine` when the
file is syntactically valid but requires security review before indexing.

## 3. Quarantine lifecycle

1. Authenticate user and tenant membership.
2. Reserve tenant quota with an idempotency key.
3. Stream bytes to quarantine storage using a server-generated object key.
4. Hash the object and record content length before parser dispatch.
5. Run allowlist, magic-byte, path traversal, archive bomb, and size checks.
6. Dispatch only accepted or review-required objects to the converter sandbox.
7. Keep quarantined originals for 14 days by default, unless legal hold extends
   retention.
8. Delete sandbox workdirs after every success, rejection, timeout, or crash.

Uploaded filenames are metadata only. They are never used as filesystem paths,
archive extraction destinations, object keys, shell arguments, or user-visible
download names without escaping.

## 4. Sandbox profile

The structured sandbox policy is
`bench/markhand_web/security/sandbox-profile.json`. Required controls:

- Non-root UID/GID (`uidMin >= 10000`, `gidMin >= 10000`, `nonRoot=true`).
- Read-only root filesystem.
- Writable tmpfs work directory only.
- No host path mounts.
- No egress, no ingress, no DNS, and no loopback networking.
- Drop all Linux capabilities and set no-new-privileges.
- CPU, RAM, file, process, open-file, archive, output, and wall limits.
- Process-group kill on timeout or worker cancellation.
- Redacted logs; uploaded text is never emitted to gate logs.

Policy/sandbox smoke evidence is generated by
`python3 bench/markhand_web/scripts/run_upload_security.py`. The harness performs
in-process denial simulations for egress, traversal, fork bomb, and timeout. It
does not claim container runtime enforcement or malware scanner coverage.

## 5. Parser and content handling

- Parser exceptions fail closed.
- PDF/OCR/native libraries must come from the runtime license inventory and
  approved requirements.
- HTML output strips scripts, style execution, event handlers, remote resources,
  and unsupported tags.
- CSV formula-leading cells are quarantined or escaped before any downstream
  display/export path.
- Converted Markdown is untrusted passage text. Retrieval/LLM prompts must frame
  it as data, not instructions.

## 6. Tenant, token, and quota controls

- Upload API checks tenant/project authorization before accepting bytes.
- Object keys are server-derived and include tenant and document-version
  namespace components.
- Workers receive only object-scoped read and write handles, never user tokens,
  provider API keys, refresh tokens, or database superuser credentials.
- Quota reservation is atomic, idempotent, and released on rejection or timeout.
- Per-tenant queue caps prevent one tenant from exhausting shared workers.
- Audit events include tenant id, document id, disposition, threat class, and
  redacted reason code.

## 7. Adversarial disposition policy

Every fixture in `bench/markhand_web/adversarial/manifest.json` must have a
matching entry in
`bench/markhand_web/security/adversarial-disposition.json`. Dispositions are:

- `reject`: do not parse/index; return user-safe error.
- `quarantine`: preserve object for review; do not index until a later policy
  explicitly promotes it.
- `allow`: only for benign controls; not used by the P0-09 adversarial set.

The P0-09 fixtures intentionally map to `reject` or `quarantine`.

## 8. Runtime license policy

- Runtime native libraries and models must be listed in both
  `docs/markhand-web-runtime-license-requirements.json` and
  `docs/markhand-web-runtime-license-inventory.json`.
- Bundled runtime entries must have approved disposition and redistribution
  allowed or source-offer-required.
- Unknown, unresolved, or unclear licenses are not bundled.
- PhoWhisper remains excluded and not bundled until a redistributable license is
  approved.
- `python3 scripts/check-runtime-license-inventory.py` is the local G0-LIC
  checker.
